### Response

After each request, Travelgate processes the data and provides a response to the Seller. Upon receiving a `HotelRatePlanInventoryNotif` request, Travelgate sends a corresponding `HotelRatePlanInventoryNotif` response. The response options include either **success** or an **error**.

**Success:**

```html
<HotelRatePlanInventoryNotifResponse xmlns="http://schemas.xmltravelgate.com/hubpush/provider/2012/10">
    <HotelRatePlanInventoryNotifResult>
        <Success xmlns="http://www.opentravel.org/OTA/2003/05"/>
    </HotelRatePlanInventoryNotifResult>
</HotelRatePlanInventoryNotifResponse>
```

**Error:**

```html
<HotelRatePlanInventoryNotifResponse xmlns="http://schemas.xmltravelgate.com/hubpush/provider/2012/10">
    <HotelRatePlanInventoryNotifResult>
        <Errors xmlns="http://www.opentravel.org/OTA/2003/05">
            <Error Language="en" ShortText="Unexpected error" Code="-1">Hotel 1610 already exists in CP.</Error>
        </Errors>
    </HotelRatePlanInventoryNotifResult>
</HotelRatePlanInventoryNotifResponse>
```

## Inventory Retrieval

Method to **retrieve** the product: hotel, rate, room.

### Request

The `HotelRatePlanInventoryRetrieve` request message is sent to retrieve a list of hotels, rate plans, and rooms, along with their configurations.

:::caution

Inactive rate plans and rooms will not be returned in this message.

:::

[Añadir ejemplo]

### Response

After each request, Travelgate processes the data and provides a response to the Seller. Upon receiving a `HotelRatePlanInventoryRetrieve` request, Travelgate sends a corresponding `HotelRatePlanInventoryRetrieve` response. 

The response options include either **success** or an **error**. In the event of success, Travelgate will provide a response to the Seller  by retrieving a list of hotels, rate plans, and rooms, along with their configurations.
