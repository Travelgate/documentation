      <details style={{ pointerEvents: "none" }}>
        <summary>
          <strong>hotelConfirmationNumberStatus</strong> <em>(String)</em>
        </summary>
        Process Status in order to obtain the hotel confirmation number. If this field is null or empty that means that the status is UNKNOWN. Possible values:
        "P"   =&gt; (pending): The booking hasn't been assinged to any operator yet, or it has but the operator did not make the call yet.
        "R"   =&gt; (retrying): The operator tried to call the hotel but, for some reason, the call couldn't be completed because of external factors. This means the operator will try to call again later.
        "NA"  =&gt; (no answer): The operator tried to call the hotel several times but the hotel did not answer.
        "C"   =&gt; (cancelled): The operator has been informed by the hotel that the reservation has been cancelled.
        "DNE" =&gt; (does not exist): The operator has been informed by the hotel that the reservation does not exist.
        "F"   =&gt; (fetched): The operator called the hotel and they got the hotel confirmation number (a truly succesful call)
        "I"   =&gt; (issued): The operator called the hotel but there were issues. Maybe the hotel is closed, maybe natural disaster situations are alerted for the check-in date, etc.
      </details>

      <details style={{ pointerEvents: "none" }}>
        <summary>
          <strong>bookingID</strong> <em>(String)</em>
        </summary>
        HotelX's booking reference. The BookingID is a unique identifier for each reservation that can be used to perform Booking queries and Cancel mutations in the system.
        It is not recommended to modify or use this value externally, although it may be useful for identifying and managing reservations within the system.
        It is important to note that the BookingID value should not be interpreted in any other way beyond its use within the system.
