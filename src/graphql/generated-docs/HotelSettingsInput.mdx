
<details>
  <summary>
  **HotelSettingsInput** (*INPUT_OBJECT*)  
  Settings are the common configurations used to construct requests to the supplier/s. By default,
we apply the same configuration to all Hotel-X clients in our back system.  
  </summary>
  
  <details style={{ pointerEvents: "none" }}>

      <strong>context</strong> <em>(String)</em>
    </summary>
    Indicates the context of the I/O codes (hotel, board, room and rates)
  </details>

  <details style={{ pointerEvents: "none" }}>
    <summary>
      <strong>client</strong> <em>(ID)</em>
    </summary>
    Client name set in TGX, present only if the authorization header is of the JWT type. It is used to modify the user initially set in the preload.
  </details>

  <details style={{ pointerEvents: "none" }}>
    <summary>
      <strong>timeout</strong> <em>(Int)</em>
    </summary>
    Specifies the time limit, in milliseconds, for connecting with suppliers. If multiple suppliers are involved, this timeout applies to all of them. If a supplier exceeds this time limit, the connection to that supplier will be cut. However, Travelgate will keep the connection with the client even if this timeout is exceeded.
    If not set, the value saved in your Hotel-X API default settings will be used.
  </details>

  <details style={{ pointerEvents: "none" }}>
    <summary>
      <strong>operationTimeout</strong> <em>(Int)</em>
    </summary>
    Defines the maximum time, in milliseconds, before an operation is cancelled by our system. Once this time is reached, the operation will be aborted and an error will be returned. This timeout is for the Hotel-X layer and should be set to a higher value than the timeout for suppliers. For example, you could specify timeout as 20000ms and operationTimeout as 22000ms. This means that we will send a timeout of 20000ms to the suppliers and will close the connection at 22000ms if the operationTimeout is exceeded. Optional.
    @deprecated(reason: "deprecated from 2024-11-11.")
  </details>

            <li><code>BALEARIC_RESIDENT</code>: BALEARIC_RESIDENT is applicable to Balearic Islands residents only.</li>
            <li><code>LARGE_FAMILY</code>: LARGE_FAMILY is applicable to large families and is determined by each supplier.</li>
            <li><code>HONEYMOON</code>: HONEYMOON is applicable to those who just got married and is determined by each supplier.</li>
            <li><code>PUBLIC_SERVANT</code>: PUBLIC_SERVANT is applicable to public servants only.</li>
            <li><code>UNEMPLOYED</code>: UNEMPLOYED applies to individuals who are without work.</li>
            <li><code>NON_REFUNDABLE</code>: NON_REFUNDABLE is applicable to non refundable options.</li>
            <li><code>MOBILE</code>: MOBILE are special rates confirmed by mobile phone.</li>

            <li><code>ESSENTIAL_WORKER</code>: ESSENTIAL_WORKER are special hotel rates for essential workers.</li>

    </summary>
    Array of plugins to execute.

    <details open>
      <summary>
        <strong>step</strong> <em>(PluginStepType)</em>
      </summary>
      Indicates where the plugin will be executed. @deprecated(reason: "deprecated from 2025-08-25. This property is no longer recommended for use.")
      <strong>Possible values:</strong>
      <ul>
        <li><code>REQUEST</code>: Plugins executed after the Buyer requests a message to Hotel-X</li>
        <li><code>REQUEST_ACCESS</code>: Plugins executed before sending a request to the supplier using access, and after the Accesses have been calculated</li>
        <li><code>RESPONSE_OPTION</code>: Plugins executed after the supplier responds a message, applied to every option they return</li>
        <li><code>RESPONSE_ACCESS</code>: Plugins executed after all access options have been responded to</li>
        <li><code>RESPONSE</code>: Plugins executed before Hotel-X responds the message to the Buyer</li>
      </ul>
    </details>

    <details>
      <summary>
        <strong>pluginsType</strong> <em>(PluginsInput)</em>
      </summary>
      Indicates the plugin that will be executed.

      <details open>
        <summary>
          <strong>type</strong> <em>(PluginType)</em>
        </summary>
        Type of the plugin to execute. @deprecated(reason: "deprecated from 2025-08-25. This property is no longer recommended for use.")
        <strong>Possible values:</strong>
        <ul>
          <li><code>PRE_STEP</code>: PRE_STEP is the first plugin that will be executed, allowing a full range of operations:
splitting arrays, joining arrays, modifying object values, and adding or removing object instances</li>
          <li><code>HOTEL_MAP</code>: HOTEL_MAP allows to match Seller and Buyer hotel codes based on contexts</li>
          <li><code>BOARD_MAP</code>: BOARD_MAP allows to match Seller and Buyer board codes based on contexts</li>
          <li><code>ROOM_MAP</code>: ROOM_MAP allows to match Seller and Buyer room codes based on contexts</li>
          <li><code>CURRENCY_CONVERSION</code>: CURRENCY_CONVERSION allows to apply currency conversion over all the prices in one Option</li>
          <li><code>MARKUP</code>: MARKUP allows the application of markup over the price</li>
          <li><code>AGGREGATION</code>: AGGREGATION allows aggregating multiple supplier options.</li>
          <li><code>POST_STEP</code>: POST_STEP is the last plugin that will be executed, allowing a full range of operations:
split arrays, join arrays, modify object values, add or remove object instances</li>
          <li><code>COMMISSION</code>: COMMISSION plugin allows the user to calculate the commission</li>
          <li><code>MARKETS_GROUP</code>: MARKETS_GROUP plugin allows the user to group search result by markets (SO3166_1_alfa_2) with shared options</li>
          <li><code>AMENITY_MAP</code>: AMENITY_MAP allows to match Seller and Buyer amenities codes based on contexts</li>
          <li><code>PROMOTION_MAP</code>: PROMOTION_MAP allows to match Seller and Buyer promotion codes based on contexts</li>
        </ul>
      </details>
