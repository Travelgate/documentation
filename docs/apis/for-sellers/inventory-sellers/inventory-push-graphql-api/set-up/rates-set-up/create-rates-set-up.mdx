# Create Rates

The `createRateSetUp` mutation allows you tu create new rates for a hotel in a specific client - seller relation in Inventory. The returned fields include:

* `code`
* `name`
* `hotelCode`
* `active`
* `mealPlanIncluded`
* `agePolicies`
	* `maxAgeChildren`
	* `maxAgeInfants`
	* `freeInfants`
	* `freeChildren`
* `paymentPolicies`
	* `currency`
	* `commission`
	* `priceIsBinding`
	* `acceptedPayments`
		* `type`
		* `cardTypes`
* `bookingRules`
	* `bookingWindow`
		* `start`
		* `end`
	* `markets`
		* `included`
		* `excluded`
	* `rateRule`
	* `seniorRule`
* `cancelPolicies`
	* `baseCancelPolicy`
		* `refundable`
		* `cancelPenalties`
			* `daysBeforeArrival`
			* `penaltyType`
			* `value`
	* `cancelPoliciesByDate`
		* `start`
		* `end`
		* `cancelPolicy`
			* `refundable`
			* `cancelPenalties`
				* `daysBeforeArrival`
				* `penaltyType`
				* `value`
* `surcharges`
	* `chargeType`
	* `taxType`
	* `value`
	* `applyType`
	* `perNight`
	* `perPax`
* `rooms`
	* `code`
	* `master`
		* `id`
		* `code`
		* `name`
	* `active`
	* `externalCode`
	* `standard`
	* `uses`
		* `numberOfGuests`
		* `minAge`
		* `maxAge`
		* `paxType`
