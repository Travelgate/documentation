# Logging

## Overview

Logging is a powerful monitoring feature that allows users to access and analyze transaction logs across the Travelgate platform. It is designed to track and analyze the booking flow, identify error codes, and troubleshoot performance issues. 

This document details the steps to search, extract traces, and download request logs (XML/JSON) using the Travelgate Logging API. The queries detailed below provide the exact same level of data access as our web platform.

:::info Authentication
For all Logging API calls, it is required to first retrieve the bearer token using the `admin { jwt }` query. The returned token must be included in the header of subsequent requests: `Authorization: Bearer <your_jwt>`.
:::

## Considerations & Business Rules

Before querying the logs, keep in mind the following platform rules:

* **Error Codes vs. Reservation Status:** The "Status/Error code" returned in the logs *only* indicates if the transaction was processed through our integration without technical errors. **It does not reflect the actual business status of a reservation.** To know the real status of a booking with the supplier, you must perform a [BookingRead](https://docs.travelgate.com/docs/apis/for-buyers/hotel-x-pull-buyers-api/booking-management/booking-read/) operation.
* **Search Logs Auditing:** Travelgate processes over 6 billion daily transactions, meaning **Search logs are not stored by default**. To retrieve Search logs via the API, an audit must first be triggered manually from the Web Platform (Monitoring > Logging > Audit Search Logs). Once triggered, the system stores 5% of the Search traffic for 5 minutes.
* **One Step Book:** You can successfully retrieve both Quote and Book logs for a One Step Book operation by filtering by connection, reference, or Session ID.
* **Trace Types:** Depending on the buyer's integration, you will see either **HotelX Logs** or **Legacy Logs**, but never both. **Connector Logs** (communication directly with the Seller) are always available.

## Query Overview

The Logging API offers three main methods to search and retrieve information:

### 1. getLogs
Primary query to search the log history by applying multiple filters (dates, supplier, buyer, operation type, or error codes).

### 2. getLogsByReference
Designed to search for logs using a specific locator or reference (client or supplier ID).

### 3. getQuoteLogsFromBook
Specific query to track the original Quote that generated a specific Booking.

## Examples

### Example: Search Logs (`getLogs`)

**Query**
```graphql
query getLoggings($input: LoggingLogsWhereInput!) {
  logging {
    getLogs(limit: 1000, where: $input) {
      totalCount
      adviseMessage {
        code
        description
        level
      }
      edges {
        node {
          logData {
            timestamp
            execTime
            api
            buyer { code }
            seller { code }
            access {
              accessData { code }
            }
            client {
              clientData { name code }
            }
            errorCode
            errorType
            operationCode
            operationType
            sessionID
            trafficType
            files {
              hotelX
              legacy
              connector
              connectorUnencrypted
            }
          }
        }
      }
    }
  }
}
```

**Variables**
```json
{
  "input": {
    "api": "hotel",
    "timeRange": {
      "absolute": {
        "from": "2026-04-07T09:38:00.000Z",
        "to": "2026-04-07T10:38:00.000Z"
      }
    },
    "mode": "BUYER",
    "errorCode_in": [0, 102],
    "operationType_in": ["BOOKING"],
    "buyer_in": ["BUYER_CODE"],
    "seller_in": ["SELLER_CODE"]
  }
}
```

**Response Example**
```json
{
  "data": {
    "logging": {
      "getLogs": {
        "totalCount": 1,
        "edges": [
          {
            "node": {
              "logData": {
                "timestamp": "2026-04-07T09:40:05.123Z",
                "operationType": "BOOKING",
                "sessionID": "9ffb5416-249f-4916-b9f0-88c2198804dc",
                "files": {
                  "hotelX": "[https://api.travelgate.com/logs/v1/download/](https://api.travelgate.com/logs/v1/download/)...",
                  "connector": "[https://api.travelgate.com/logs/v1/download/](https://api.travelgate.com/logs/v1/download/)..."
                }
              }
            }
          }
        ]
      }
    }
  }
}
```

## Frequently Asked Questions

<details>
    <summary>Why can't I find my error logs by reference?</summary>
    <div>Failed bookings or cancellations cannot be retrieved using locators. You must use the <strong>Time Range</strong> filter combined with the specific error code.</div>
</details>

<details>
    <summary>What date format should I use?</summary>
    <div>You must use the ISO 8601 format with enclosing double quotes: <code>"2026-04-07T09:38:00.000Z"</code>.</div>
</details>
