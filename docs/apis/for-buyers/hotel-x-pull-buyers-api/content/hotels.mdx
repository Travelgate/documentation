# Hotels

The Hotels Query allows you to retrieve hotel information in two ways:  
- The **FastX Hotel List**, a unified list of all unique hotels available to you in the Travelgate Marketplace. These FastX codes are the ones used throughout the booking flow.  
- The **Supplier Hotel List**, which provides the hotel information returned for each connected Seller.

This flexibility lets you work with both standardized FastX codes for a seamless integration and the original Supplier data when you need full visibility into each Partner’s portfolio.

The returned fields include:

* `hotelCode`
* `hotelName`
* `giataData`
* `location`
* `allAmenities`
* `medias`
* `descriptions`

## Query Overview

When creating your hotels query, you have two different inputs to fill based on your specific needs:

1. Criteria
2. Token

### 1. Criteria 

This query offers versatility, with certain fields marked as mandatory (`access`) and others as optional (`hotelCodes`, `countries`, `destinationCodes`, etc.). This flexibility empowers you to create a personalized Hotels Query, tailoring the requested fields to your specific needs. Consequently, the response will only include information that is highly relevant to you, streamlining the retrieval process and ensuring the data you receive meets your exact requirements.

```js
{
	"criteriaHotels": {
		"access": "2",
		"hotelCodes": [
			"91415"
		],
		"maxSize": 5
	}
}
```

**Mandatory criteria**
* `access`

**Optional criteria**
* `hotelCodes` (Search by hotel)
* `countries` (Search by country)
* `destinationCodes` (Search by destination)
* `maxSize` (Paginate response)


### 2. Token

The HotelX API provides paginated responses to handle cases where the number of hotels from the requested access exceeds 500. The default response page contains 500 hotels, but you can specify the number of objects returned per page using the `maxSize` field in the criteria variable explained before to retrieve between 1 and 500 hotels.

To include the token in your request, all you need to do is:

1. Add the `token` object to the fields and variables. 

2. After receiving a response, you will only receive a portion of the full list. To access the next page, simply copy the `token` tag from the response and include it in your next request. This will allow you to obtain a new set of objects in the following response.

3. Repeat these steps until the `token` is no longer included in the response, the query will return “Hotels not found”.

**Query variables**

First, in your hotels query you won't indicate any `token` input:

```js
{
    "criteriaHotels" : {
        "access" : "2",
        "maxSize" : 5
    },
    "token" : ""
}
```

You will obtain a response with 5 hotels per page:

**Response (truncated):**
```js
{
  "data": {
    "hotelX": {
      "hotels": {
        "token": "FGluY2x1ZGVfY29udGV4dF91dWlkDnF1ZXJ5VGhlbkZl...",
        "edges": [ {...}, {...} ]
      }
    }
  }
}
```

**Example — Next Page (include token):**
```js
{
  "criteriaHotels": {
    "access": "2",
    "maxSize": 5
  },
  "token": "FGluY2x1ZGVfY29udGV4dF91dWlkDnF1ZXJ5VGhlbkZl..."
}
```

## Use Cases

### FastX master hotel list

To retrieve the **FastX Hotel Master**, make a `hotel list` call using your **FastX access**, which you can find in the *MyConnections* section under the Seller **"Travelgate"**. This returns the complete list of unique hotels available to you across all connected Suppliers. As you add new Sellers, any new properties they provide — not already offered by your other connected Suppliers — are automatically included in your master list.

:::info FastX Content
When performing a Hotels Query **using the FastX access**, the following fields are currently returned:
- Hotel code
- Hotel name
- Country
- Address
- Latitude / Longitude
- GIATA ID (if this functionality has been [activated](/kb/connectivity-products/for-buyers/mapping-solutions/giata/giata-multicodes))

**Coming soon:**
To obtain complete enriched static hotel data for each FastX code, you can activate our FastX Content solution, which adds standardized content including descriptions, images, amenities, and more.
:::

### Supplier hotel list

To retrieve only one Supplier's portfolio, include that Supplier's access in the query (for example, access `2` for the Travelgate Test supplier).

The response returns the Supplier's native static content, so the available data depends on what that Seller provides. In this case, `hotelCode` corresponds to the Supplier's native hotel code, and you can also find the matched FastX code in the `mappings` field.

<GraphqlSample query={hotelsQuery} variables={hotelsVariables} />

:::note

The amount of information and content returned might vary between Sellers.

HotelX supports all languages, but receiving hotel descriptions in one or another depends entirely on the languages supported by the Seller.

:::

### Hotel list by country

When utilizing the Hotels List Query, for more personalized results, filter specific hotels, countries or destinations based on your interests. For example, to focus solely on the inventory of all your Sellers in Greece, add the `countries` field with "GR" in your criteria. This will ensure you receive hotels directly related to that country, providing you with highly targeted and relevant information.

## Frequently Asked Questions




<details>
    <summary>Can I get the total property count without fetching all hotels?</summary>
    <div>
        <div>Yes. Add the `count` field to your Hotels Query to retrieve the total number of properties. You can request `count` alone without `edges` if you only need the total.</div>
		<b>Request</b>
```graphql
query {
    hotelX {
		hotels(criteria: {access: "2"}) {
			count
			edges {
				...
			}
		}
	}
}
```		
		<b>Response</b>
```graphql
{
	"data": {
		"hotelX": {
			"hotels": {
				"count": 16,
				"edges": [
					...
				]
			}
		}
	}
}
```		
    </div>
</details>


<details>
    <summary>How can I obtain GIATA codes?</summary>
	<div>
        <div>GIATA codes are supported at the hotel level and must be activated first. **Setup steps:** </div>
		1. [Open a ticket](https://app.travelgate.com/support) with Customer Care with your GIATA credentials and the Suppliers you want to activate GIATA for.
		2. Once enabled, include the `giataData` node in your Hotels Query to receive GIATA identifiers.
		
		<b>Request</b>
		
```graphql
query {
    hotelX {
		hotels(criteria: {access: "2"}) {
			edges {
				cursor
				node {
					code
					hotelData {
						hotelCode
						hotelName
						giataData {
						updatedAt
						source
						href
						}
						...
					}
				}
			}
		}
	}
}
```

					{
						"cursor": "",

							"code": "",
							"hotelData": {
								"hotelCode": "66337",
								"hotelName": "Hotel Test",
								"giataData": "11267",
								...
							}

					}
				]
			}
		}
	}
}
