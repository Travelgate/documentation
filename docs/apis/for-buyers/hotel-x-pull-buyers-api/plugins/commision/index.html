Commission

Commission plugin defines the contracted/expected commission in cases where the Supplier is not able to send it via integration. The commission loaded in the FTP is considered as a Gross commission, meaning that the commission or profit is already included in the price.

To get started with the Commission plugin, follow these simple steps:

1. Create and Upload your Commission Fileâ€‹

To get started, create your commission file in the correct format and then upload it to your FTP account. We've provided a template file below â¬‡ï¸ to help you with this process. Just click on the file name, add your own rules, save it in CSV format, and then upload it to the corresponding folder in your FTP.

File Format Specificationâ€‹

  • File Name: BuyerContext_sequential_commission.csv
  • Directory: F0_0000/HotelX0000/BusinessRules
  • Delimiter: Comma (,)
  • Header Row:
    • RuleId (mandatory): Unique identifier of the rule (assign one)
    • ClientTokens: Client token used for index the rule
    • SupplierCodes (mandatory): List of the supplier codes you want the commission to be applied to.
      • Separator for multiples codes in the same row: Semicolon (â€œ;â€)
    • NotSupplierCodes (mandatory): List of the supplier codes you don't want the commission to be applied to.
      • Separator for multiples codes in the same row: Semicolon (â€œ;â€)
    • CreationDateFrom: Start creation date (yyyy-mm-dd)
    • CreationDateTo: End creation date (yyyy-mm-dd)
    • CheckInFrom: Checkin date (yyyy-mm-dd)
    • CheckInTo: Checkout date (yyyy-mm-dd)
    • HotelCodes: List of the hotel codes you want the commission to be applied to.
      • Separator for multiples codes in the same row: Semicolon (â€œ;â€)
    • NothotelCodes: List of the hotel codes you don't want the commission to be applied to.
      • Separator for multiples codes in the same row: Semicolon (â€œ;â€)
    • ChainsCodes: List of the chain codes you want the commission to be applied to.
      • Separator for multiples codes in the same row: Semicolon (â€œ;â€)
    • NotChainCodes: List of the chain codes you don't want the commission to be applied to.
      • Separator for multiples codes in the same row: Semicolon (â€œ;â€)
    • DestinationCodes: List of the destination codes you want the commission to be applied to.
      • Separator for multiples codes in the same row: Semicolon (â€œ;â€)
    • NotDestinationCodes: List of the destination codes you don't want the commission to be applied to.
      • Separator for multiples codes in the same row: Semicolon (â€œ;â€)
    • Percentage (mandatory): Commission value (decimal separator must be point (â€œ.â€))
    • Force (mandatory): Boolean value true/false
      • True: The rule will be applied if matching all its conditions criteria, regardless of a commission being received or not from the supplier through the integration.
      • False: The rule will not be applied despite of matching all its conditions criteria if a commission is received from the supplier through the integration.
